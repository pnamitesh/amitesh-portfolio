The Titan Dashboard Redesign

UX/UI Design | Front-End Development | Data Visualization

This project involved a complete overhaul of an outdated internal data dashboard used by 500+ operations employees. The goal was to improve data legibility, streamline workflows, and reduce the average time spent on daily reporting tasks by 25%.

My Role

Lead UX Designer & Front-End Developer

Duration

8 Weeks (Concept to Launch)

Tools Used

Figma, React, SASS, D3.js

1. The Problem

The legacy dashboard was cluttered, unresponsive, and lacked key filtering capabilities. User interviews confirmed that the confusing layout led to significant errors in data interpretation and an average of 45 minutes wasted per day in manual cross-referencing between systems.

**Core Challenge:** Reduce cognitive load while integrating data from three disparate APIs into one cohesive view.

2. The Solution & Process

I employed a user-centered design approach, starting with wireframing and low-fidelity prototypes in Figma. The solution involved implementing a modular card-based UI powered by **React**. Key features included a unified search bar and dynamic real-time charts using **D3.js** for immediate data comprehension.

Specific steps:

  • Conducted 10 stakeholder interviews to map existing pain points.
  • Created a component library in Storybook for future scalability.
  • Developed a dedicated mobile view for on-the-go reporting.

3. Key Results

The new dashboard was rolled out to the operations team with positive feedback. The metrics confirm the success of the redesign:

  • **28% Reduction** in the average time required to complete the daily reporting task (surpassing the 25% goal).
  • **95% User Satisfaction** rating in a post-launch survey.
  • **75% Decrease** in reported data entry errors within the first month.
